## Описание проекта

REST-API сервис управления иерархическими группами людей.

Группы образуют дерево (у группы могут быть дочерние группы), люди привязаны к
группам. Сервис умеет считать участников как в самой группе, так и рекурсивно
вместе с дочерними, а также выводить списки людей по тем же двум режимам.

### Технологический стек

- **Go 1.25**
- **PostgreSQL 16** (`pgx/v5`)
- **REST** + контракт **OpenAPI 3.0** (`oapi-codegen`), роутер `chi`
- Миграции — `golang-migrate` (с тестовыми данными)
- Docker + docker-compose

## Запуск проекта

### Подготовительные действия

1. Установить [Docker Desktop](https://www.docker.com/products/docker-desktop/) (включает Docker Engine, Compose V2 и WSL2-бэкенд)
2. Запустить Docker Desktop и дождаться статуса *Running*

### Настройка секретов

Конфигурация задаётся через переменные окружения. В репозитории есть шаблон
[`.env.example`](.env.example) со всеми параметрами и значениями по умолчанию —
скопируйте его в `.env` (файл `.env` в `.gitignore` и не попадает в репозиторий):

```sh
cp .env.example .env
```

docker-compose автоматически подхватывает `.env` из корня проекта. Все переменные
имеют значения по умолчанию, поэтому для локального запуска `.env` можно не
создавать.

| Переменная | Назначение | Значение по умолчанию |
|------------|-----------|------------------------|
| `POSTGRES_USER` | Пользователь PostgreSQL | `postgres` |
| `POSTGRES_PASSWORD` | Пароль PostgreSQL | `postgres` |
| `POSTGRES_DB` | Имя базы данных | `groups` |
| `POSTGRES_PORT` | Порт PostgreSQL на хосте | `5432` |
| `DB_DSN` | Строка подключения к БД (для локального запуска без compose) | `postgres://postgres:postgres@postgres:5432/groups?sslmode=disable` |
| `HTTP_HOST` | Адрес, на котором слушает HTTP-сервер | `0.0.0.0` |
| `HTTP_PORT` | Порт HTTP-сервера | `8080` |
| `LOG_LEVEL` | Уровень логирования: `debug` \| `info` \| `warn` \| `error` | `info` |

> При смене `POSTGRES_USER`/`POSTGRES_PASSWORD`/`POSTGRES_DB` строка `DB_DSN` для
> сервисов `migrate` и `app` собирается из этих же переменных автоматически —
> отдельно править её в compose не нужно.

### Запуск приложения

```sh
docker compose -f deployments/docker-compose.yml up --build -d
```

Поднимает PostgreSQL, применяет миграции и запускает приложение.
